### Method 2: pdfjs-dist with Dynamic Import (NEW) ‚úÖ

#### How It Works:
```javascript
// No static import! ‚úÖ

async parsePDF(buffer) {
  // Load library only when needed
  const pdfjsLib = await import('pdfjs-dist/legacy/build/pdf.mjs')
  
  // Load document
  const loadingTask = pdfjsLib.getDocument({
    data: new Uint8Array(buffer),
    useSystemFonts: true,
    standardFontDataUrl: 'https://cdn.jsdelivr.net/npm/pdfjs-dist@3.11.174/standard_fonts/'
  })
  
  const pdfDocument = await loadingTask.promise
  const numPages = pdfDocument.numPages
  const pageTexts = []
  
  // Extract text from EACH page separately
  for (let pageNum = 1; pageNum <= numPages; pageNum++) {
    const page = await pdfDocument.getPage(pageNum)
    const textContent = await page.getTextContent()
    const pageText = textContent.items.map(item => item.str).join(' ')
    pageTexts.push(pageText)
  }
  
  return {
    text: pageTexts.join('\n\n'),
    pages: numPages,
    pageTexts,  // ‚úÖ Array with text per page
    metadata: { numPages, extractedAt: new Date().toISOString() }
  }
}
```

#### Pros:
- ‚úÖ **Works in serverless** (no initialization errors)
- ‚úÖ **Page-by-page extraction** (better structure)
- ‚úÖ **Higher accuracy** (preserves text positioning)
- ‚úÖ **Actively maintained** (Mozilla/PDF.js project)
- ‚úÖ **Used by millions** (Firefox's PDF viewer)
- ‚úÖ **Better error handling** (async loading)
- ‚úÖ **Configurable** (fonts, encoding, etc.)
- ‚úÖ **No file system access** during import

#### Cons:
- ‚ö†Ô∏è More code (but more control)
- ‚ö†Ô∏è Slightly slower cold start (dynamic import overhead)
- ‚ö†Ô∏è Need to handle async loading
